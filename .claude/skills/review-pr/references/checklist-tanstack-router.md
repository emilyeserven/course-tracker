# TanStack Router Checklist

- **`createFileRoute`**: Routes use `createFileRoute` with the correct path string matching the file's location in `packages/client/src/routes/`
- **Error and pending components**: Every route should define `errorComponent` and `pendingComponent`
- **Type-safe navigation**: Use `<Link to="..." params={...}>` with type-safe params — not raw `<a>` tags for internal links
- **`from` prop**: Use the `from` prop on `<Link>` for relative params resolution
- **Do not edit `routeTree.gen.ts`**: It's auto-generated by `@tanstack/router-plugin` in the Vite config
- **Auth guards and shared data**: Use `beforeLoad` and route context — not layout-level effects
- **Code splitting**: Use `lazy` route definitions for non-critical routes
- **Search params**: Type-safe search params via `validateSearch` with Zod schemas — not raw URL parsing
- **Loaders for data fetching**: Prefer route `loader` functions over `useEffect` + `fetch` to avoid waterfalls
- **Use Context7**: Verify route API usage against current TanStack Router docs when reviewing route changes
